# Resolve Question Shape

Default Resolve questions should be compact and decision-ready.

## Preferred structure

1. **Question**
2. **Why it matters**
3. **Default recommendation**
4. **What remains blocked or fuzzy without the answer**

## Example skeleton

- **Question:** Should the canonical home be `docs/directives/` or a nested subtree?
- **Why it matters:** This changes routing law and the contract-family docs.
- **Default recommendation:** `docs/directives/` as a top-level family.
- **Without this answer:** the docs pack can still be drafted, but it risks hardening the wrong routing doctrine.

## Keep it brief

Resolve questions should normally be one short executive brief, not a menu of fake options.

If one path is clearly better, say so.

## One at a time

Do not ask the next Resolve question until the previous answer has been routed back into the relevant live lane or canonical artifact.
