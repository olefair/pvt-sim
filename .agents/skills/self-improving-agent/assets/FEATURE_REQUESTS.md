# Feature Requests

Missing capabilities, missing automation, or repeated user asks that should
become backlog items or future improvements.

Do not copy private prompts, secrets, or unnecessary personal data into feature
request entries.

Use entry IDs like `FEAT-YYYYMMDD-XXX`.

Suggested statuses:

- `pending`
- `in_progress`
- `resolved`
- `wont_fix`
- `promoted_to_skill`

Capture:

- the requested capability
- why it mattered to the user or workflow
- a rough complexity estimate
- the best known implementation direction
