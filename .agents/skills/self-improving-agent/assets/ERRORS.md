# Errors

Unexpected command, tool, runtime, or integration failures captured for future
prevention.

Redact secrets, tokens, credentials, and oversized sensitive logs before saving
entries here.

Use entry IDs like `ERR-YYYYMMDD-XXX`.

Suggested statuses:

- `pending`
- `in_progress`
- `resolved`
- `wont_fix`

Keep each entry concrete:

- what failed
- what the actual error was
- enough context to reproduce or recognize it again
- the best known fix or next diagnostic step
