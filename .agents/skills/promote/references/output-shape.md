# Promote Output Shape

Default Promote output should be short and operational.

## Default sections

- **Reviewed**
- **Promoted to Blueprint**
- **Promoted to Plan**
- **Direct execution / config route**
- **Remain as Task**
- **Questions / ambiguities**

## Style rules

- Prefer one coherent summary over noisy per-task chatter.
- Default to **preview-first** unless Ole explicitly wants direct write-through.
- Say clearly whether the pass was **preview-first** or **write-through**.
- If you merged tasks, state that and preserve lineage in the artifacts.
- Omit empty sections unless the user asked for a full audit-style dump.

## If preview-first

Make the preview decision-useful:

- what would be created
- what would be merged
- what would stay light
- what still needs Resolve before promotion
