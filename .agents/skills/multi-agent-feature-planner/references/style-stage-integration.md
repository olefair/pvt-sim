# Style Stage Integration

This workflow is built on the original style chain. Do not treat those style skills as decorative references. Treat them as the cognitive basis for the worker stages.

## Stage mapping

### Generation = Explore-derived

Primary inheritance:
- `style-explore`

Carry forward:
- widen the problem space
- branch into multiple framings
- surface tensions and tradeoffs
- resist premature convergence
- avoid action-plan closure

Adaptation for this workflow:
- each worker writes an independent domain paper
- the worker may notice weak assumptions or failure modes in its own idea
- the worker does **not** argue with absent peers yet
- the worker should not collapse into recommendation-only writing too early

### Review = Reflect-derived

Primary inheritance:
- `style-reflect`

Carry forward:
- mirror what was actually said
- organize themes, tensions, open questions, and promising directions
- avoid injecting brand-new ideas where possible
- compress faithfully before judging

Adaptation for this workflow:
- each worker reads the other papers individually
- this is peer review, not debate
- workers should internalize the holistic shape of the other proposals before attacking them
- workers may note tensions, overstatements, and possible failure modes, but should not yet turn them into live adversarial exchange

### Debate = Reflect + Show Working hybrid

Primary inheritance:
- `style-reflect`
- `style-show-working`

Carry forward from Reflect:
- represent the strongest version of the others' ideas accurately
- identify unresolved contradictions and genuine tensions

Carry forward from Show Working:
- make reasoning explicit
- show why an approach would work in practice
- surface constraints, file surfaces, risks, and implementation implications before claiming a direction is viable

Adaptation for this workflow:
- debate is not pure rhetoric
- workers should pressure-test both their own ideas and others' ideas against practical reality
- workers should aim to prove viability, not just score points
- the goal is stronger coalescence or a clean deadlock, not endless controversy

### Plan synthesis = Plan-derived

Primary inheritance:
- `style-plan`

Carry forward:
- organize goals, phases, deliverables, dependencies, and risks
- stay strategic, not implementation-level
- convert synthesized understanding into structured planning artifacts

### Handoff / pre-build gate = Show Working-derived

Primary inheritance:
- `style-show-working`

Carry forward:
- verify that the resulting plan/package is implementation-ready
- restate the intended build task clearly
- identify issues and file surfaces before builder execution begins

## Negative guidance

- Do not let Generation collapse into Plan.
- Do not let Review collapse into Debate.
- Do not let Debate become freeform contradiction theater.
- Do not let Plan synthesis drift into Build.
- Do not let Show Working become implementation itself.
