# Progress Output Shape

Default Progress output should be short, executive, and decision-useful.

## Default sections

- **Active**
- **Blocked / waiting**
- **Waiting on Ole**
- **Next up**
- **Can be closed**
- **Parked / deferred**

## Style rules

- Keep it concise.
- Lead with the real bottleneck.
- Prefer one coherent answer over fragmented mini-updates.
- If a section is empty, omit it unless the user asked for a full template-style rundown.

## Waiting on Ole brief shape

When relevant, format each item as:

- decision needed
- why it matters
- default recommended action
- what remains blocked or churn-prone without the answer
- fork type

## Final synthesis line

Use a one-line synthesis when it helps, for example:

- `2 true active workstreams, 1 decision waiting on Ole, 2 resolved lanes ready to close.`

Avoid fake precision when the state is genuinely fuzzy.
