# Read Repo Output Pattern

Use this pattern when another skill needs a quick, chainable repository orientation.

## Minimum useful output

- Scope
- Task focus
- Likely entrypoints
- Relevant files
- Observed structure
- Likely touchpoints
- Unknowns / next best reads

## Notes

- Keep it concise.
- Prefer concrete files over abstract architecture talk.
- Surface uncertainty explicitly instead of smoothing it over.
